# design-diff

Локальный инструмент для визуального сравнения референс-дизайна (URL или загруженное изображение)
с целевым сайтом по нескольким брейкпоинтам. Находит пиксельные различия, зоны их концентрации
и (для пары URL↔URL) расхождения в цветах/типографике/отступах/layout. По итогам генерирует
готовый промпт для Claude — полностью локально, без обращения к каким-либо внешним API.

Всё выполняется локально. Единственный сетевой доступ — Playwright открывает страницы по URL,
которые вы сами указали, чтобы сделать скриншоты.

## Стек

- Backend: Node 20 + Express + TypeScript (запуск через `tsx`, без шага сборки)
- Скриншоты: Playwright (Chromium)
- Обработка изображений и диффы: `sharp` (все операции с изображениями) + `pixelmatch` (сырой RGBA-дифф)
- Frontend: Vue 3 + Vite + TypeScript + Tailwind CSS 3, тёмная тема
- Валидация: `zod`, загрузка файлов: `multer`

## Установка и запуск

```bash
cd "D:\My Projects\design-diff"
npm install
npm run install:browser   # ставит/проверяет Chromium для Playwright (обычно уже закэширован)
npm run dev                # поднимает Vite (5173) и Express (3000) параллельно
```

Откройте http://localhost:5173 — интерфейс проксирует `/api`, `/runs`, `/fixtures` на Express (порт 3000).

### Продакшн-сборка

```bash
npm run build   # собирает web/dist
npm start        # собирает и поднимает Express, который отдаёт web/dist и API на одном порту (3000)
```

## Как пользоваться

1. Выберите источник референса — URL другого сайта или загрузка изображения макета (PNG/JPG).
2. Укажите URL целевого сайта, который нужно сверить с референсом.
3. Выберите брейкпоинты (пресеты 1440/1024/768/390 или добавьте свою ширину).
4. При необходимости включите режим "полная страница" (fullPage) — тогда перед снимком страница
   проскроллится до конца и обратно, чтобы подгрузить ленивый контент.
5. Нажмите «Сравнить». Для каждого брейкпоинта появится карточка: оценка совпадения, переключатель
   Side-by-side / Overlay (слайдер до/после) / Diff, таблица расхождений стилей (только для пары
   URL↔URL) и зоны концентрации различий поверх скриншота цели.
6. Внизу — готовый промпт для Claude: можно скопировать в буфер или сохранить как `.md`-файл
   и вставить в Claude, чтобы он поправил вёрстку.

## Локальные тестовые фикстуры

`fixtures/a.html` и `fixtures/b.html` — две сознательно различающиеся страницы (цвет, шрифт h1,
отступы кнопки, ширина контейнера) с одинаковой структурой (header/nav/main/section/footer).
Express отдаёт их по адресам `http://localhost:3000/fixtures/a.html` и `/fixtures/b.html` —
удобно для офлайн-проверки пайплайна без выхода в интернет.

## Известные ограничения

- Hot-регионы считаются по сетке 6×4 (доля "красных" пикселей pixelmatch в ячейке), а не через
  честное распознавание связных областей — компромисс ради простоты и скорости
  (см. `// shortcut:` в `server/diff.ts`).
- Тальи палитры/типографики/отступов — наивный подсчёт по количеству элементов
  (`server/styles.ts`), не по видимой площади на экране.
- Style-diff (цвета/шрифты/отступы/layout) считается только для пары URL↔URL — для референса-
  изображения такого DOM просто не существует, поэтому используются только визуальный диф и
  hot-регионы.
- Промпт для Claude собирается по детерминированному шаблону локально — никакого обращения
  к Claude API нет.
